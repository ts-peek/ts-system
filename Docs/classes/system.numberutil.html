    Gets whether the first number equals the second number. The two values are being considered equal if the first 3 digits of their difference are all zero. If both values are undefined or null they are also considered equal. However, if one value is undefined and the other value is null, this method returns false.

    Parameters

Static equalsNDigitsPrecision

  • equalsNDigitsPrecision(first: number, second: number, precisionDigits?: number): boolean
    • Defined in System/NumberUtil.ts:73

    Gets whether the first number equals the second number. The two values are being considered equal if the first n digits of their difference are all zero. If both values are undefined or null they are also considered equal. However, if one value is undefined and the other value is null, this method returns false.

    throws

    ArgumentOutOfRangeException If the precisionDigits argument is negative.

    Parameters

    • first: number

      The first number to compare.

    • second: number

      The second number to compare.

    • Default value precisionDigits: number = 10

      The number of decimal digits that must be zero in the absolute difference |{first} - {second}| in order to be considered equal. If precisionDigits is zero, the two values must match exactly to be treated equal. Must be non-negative and less than or equal to 20. Defaults to 10.

    Returns boolean

    true, if the the two numbers are equal (with respect to the given precisionDigits argument); false otherwise.

Static isParsableAsNumber

  • isParsableAsNumber(value: string): boolean
    • Defined in System/NumberUtil.ts:106

    Checks whether or not the given value is parsable as a number. This method is more restrictive than the parseInt method provided by the JavaScript language itself. Following values cause the method to return false:

    NumberUtil.isParsableAsNumber(undefined); NumberUtil.isParsableAsNumber(null); NumberUtil.isParsableAsNumber("NaN"); NumberUtil.isParsableAsNumber("123Test"); NumberUtil.isParsableAsNumber("123,456");

    The method does not consider any cultural differences of number representations; it uses the english format without any group separators. It supports the scientific notation, i.e 1.23456E3 is considered a parsable number.

    see

    NumberUtil.parseNumber

    Parameters

    • value: string

      The value being checked.

    Returns boolean

    true if the given value may be parsed as a number; false otherwise.
